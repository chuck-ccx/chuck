# AUTONOMY.md — Chuck's Autonomy Ladder

## Levels
- **Level 0** — Fully autonomous: read-only, drafts, internal posts, analytics, memory writes
- **Level 1** — Soft approval: external comms, small DB writes, outreach sends
- **Level 2** — Hard approval: code changes, pricing, migrations, mass actions

## Per-Area Autonomy (Phase 1)

### Support
- L0: Draft replies, classify tickets, FAQ search
- L1: Send replies to customers
- L2: n/a in Phase 1

### BD / Outreach
- L0: Research, list building, draft emails
- L1: Send outreach emails, iMessage follow-ups
- L2: n/a in Phase 1

### Analytics
- L0: All read-only queries, generate reports, post internally
- L1: n/a
- L2: n/a

### Product
- L0: Create Linear/GitHub issues, draft PRDs, comment on issues
- L1: n/a
- L2: Code patches, DB migrations, deploy triggers

### Code / Infra
- L0: Read repo files
- L1: n/a
- L2: All writes — disabled in Phase 1

## Promotion Criteria
To move a task from L1 → L0:
1. 10+ successful approvals with zero rollbacks
2. Dylan explicitly promotes it
3. Logged in this file with date
